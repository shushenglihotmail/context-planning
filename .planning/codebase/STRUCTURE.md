# Folder Structure

## Top-level layout

```
.
├── bin/                CLI entrypoint(s)
├── commands/           slash-command markdown bodies installed into harnesses
├── docs/               reference documentation and generated notes
├── install/            harness installers (copilot/claude/cursor/aider)
├── lib/                runtime/state-layer modules
├── templates/          seed files used by init/scaffold commands
├── test/               flat Node test files
├── .planning/          local state for this repo's own cp workflow
├── package.json        scripts, bin mapping, metadata
└── README.md           project overview
```

## Folder responsibilities

- `bin/` — only CLI dispatch and formatting. Do not put reusable workflow logic here.
- `lib/` — pure state operations, file IO, and action-list builders. No argv parsing, no harness installation, no direct `console.log`.
- `install/` — harness-specific install logic only. Export `install({pluginRoot, repoRoot, force})`; share `install/common.js` helpers.
- `commands/cp/` — user-facing slash command bodies; keep them as markdown, not code.
- `templates/` — canonical file scaffolds copied by `cp init` and scaffold commands.
- `test/` — one file per behavior area. Keep them runnable with `node test/<file>.js`.
- `.planning/` — repo-local state and generated planning artifacts. Treat it as writable state, not application source.

## Where do I put a NEW lifecycle op?

Use the canonical pattern from `lib/inbox.js` (v0.4.0) or `lib/worktree.js` (v0.4.3):
- add `lib/<name>.js` with a pure, action-list-shaped API
- add `bin/cp.js cmd<Name>` and wire it in `main()`
- add `commands/cp/<name>.md`
- add `test/unit-<name>.js`
- register `unit-<name>.js` in `package.json` `scripts.test`
- update `usage()` text in `bin/cp.js`

## Where do I put a NEW harness installer?

Create `install/<name>.js` exporting `install({pluginRoot, repoRoot, force})`. Use `writeFileSafe` from `install/common.js`. Installer pickup is automatic via `bin/cp.js cmdInstall`; no extra switch wiring beyond the harness name.

## Filename conventions

- Files: kebab-case.
- Functions: camelCase.
- Classes: none; prefer plain functions and objects.

## Drift defense modules (v0.8)

- `lib/audit.js` — read-only consistency checker. `runAudit(root, opts)`
  returns `{ findings, summary }`. Nine built-in checks registered in
  `CHECKS`. Reuses `milestone._extractExpectedKeyFiles`,
  `state.regenerate({ dryRun: true })`, `git.shaExists`. CLI wrapper:
  `bin/commands/audit.js`. See `.planning/phases/24-cplan-audit-detection/`.
- `lib/audit-fix.js` — `cp audit --fix` orchestrator. Pluggable `FIXERS`
  registry partitioned into auto/manual/skip via `classify()`, applied
  via `applyFixes()` with one atomic commit per fix. Built-in fixers:
  `state-stale`, `summary-without-tick`, `missing-base-commit`,
  `missing-end-commit`. CLI: `bin/commands/audit.js` `--fix` branch.
  See `.planning/phases/25-cplan-audit-fix-loop/`.
- `lib/reconcile.js` — repair operations for SHA backfill and
  expected-key-files rewrite. Exports `inferBaseCommit`, `inferEndCommit`,
  `acceptExpectedKeyFiles`, and `reconcileFinding` (dispatch by finding
  id). CLI: `bin/commands/reconcile.js`. Plugged into `audit-fix.FIXERS`
  for `missing-base-commit` / `missing-end-commit`. See
  `.planning/phases/26-repair-commands/`.
- `lib/lifecycle.js::supersedePlan` — marks a plan `[~]` and appends a
  "Superseded by" note (v0.8 P10). CLI: `bin/commands/supersede.js`.
- `lib/lifecycle.js::recordDeviation` — appends a dated `## Deviation`
  block to phase PLAN.md (v0.8 P10). CLI: `bin/commands/deviate.js`.
- `lib/lifecycle.js::scaffoldPhase` — `continueFromPrior` option bypasses
  the prior-summary gate (distinct from `--force`) and stamps a
  "Continues from phase N-1" note in the new PLAN.md (v0.8 P10).
- `lib/lifecycle.js::_priorPhaseAudit` — phase-22 gate that blocks
  `scaffold-phase N` when phase N-1 has ticked plans missing SUMMARYs.
- `lib/lifecycle.js::completeMilestone` — phase-23 audit gate; refuses
  on HIGH (always) and MEDIUM (unless `--audit-warn`); `--no-audit`
  bypasses with mandatory stderr override.
- `lib/milestone.js` — owns SHA-pinned base/end commits on PLAN.md /
  SUMMARY.md (v0.8 P1) and the expected-key-files contract.
- `lib/state.js::regenerate` — derived STATE.md (v0.8 P4); the audit
  `state-stale` check piggybacks on `dryRun` mode.

## Generated / vendored / ignored

- `.planning/codebase/*` is autogenerated by `/cp-map-codebase`; do not hand-edit it.
- `.planning/` is gitignored in cp's own repo; it is local state only.
